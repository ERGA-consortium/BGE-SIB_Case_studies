# SSRseq genotyping
Turning the read files into genotypes requires the recognition of sequences belonging to each locus (based on the forward and reverse primer sequences contained at the beginning and end of the merged reads), the detection of from 1 up to 2n (= ploidy level) frequent variants of these, which would represent different alleles, and creating statistic files, including a genotype table stating each samples' detected genotype at all loci, ready for further processing by analysis software. We include here firstly a minor update (bugfix) to a pre-existing pipeline, described in [Lepais et al. 2019](https://doi.org/10.7717/peerj.9085) and available from [here](https://doi.org/10.15454/HBXKVA). Secondly, we include an extension to this pipeline, implementing probabilistic scoring of alleles/genotypes (including allele dosage information in polyploids) and a variety of genotype table formats under different allele naming conventions (e.g. easing comparison with pre-existing SSR length data), developed by Yuwei NAN in her BSc thesis.  

## Prerequisites
Apart from the demultiplexed data, this step needs the sequences of the F + R primers for each locus, expected lengths / numbers of repeats for each locus and the ploidy level of the studied organism. Both script-based pipelines need an installation of FDStools, available from [here](https://www.fdstools.nl), and can only be executed under Linux, but may be run on laptop/desktop computers.
